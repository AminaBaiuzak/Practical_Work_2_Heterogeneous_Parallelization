# Практическая работа №2
Контрольные вопросы - ответы

**1. В чём основные отличия алгоритмов сортировки пузырьком, выбором и вставкой?**

Алгоритмы сортировки пузырьком, выбором и вставкой отличаются принципом работы и характером операций.
- Сортировка пузырьком основана на многократном сравнении соседних элементов и их обмене, если они находятся в неправильном порядке. Алгоритм выполняет большое количество сравнений и обменов.
- Сортировка выбором на каждом шаге ищет минимальный элемент в неотсортированной части массива и помещает его на правильную позицию. Количество обменов меньше, чем в пузырьковой сортировке.
-	Сортировка вставкой постепенно формирует отсортированную часть массива, вставляя каждый новый элемент в нужное место. Она эффективна для небольших или частично отсортированных массивов.

**2. Почему параллельная реализация сортировки вставкой сложнее для выполнения с использованием OpenMP?**

Сортировка вставкой имеет последовательную природу, так как каждый следующий шаг алгоритма зависит от результата предыдущего. Элемент нельзя корректно вставить, пока не завершена вставка предыдущего элемента. Из-за этой зависимости между итерациями трудно эффективно разделить работу между потоками, что существенно ограничивает возможности параллелизации с использованием OpenMP.

**3. Какие директивы OpenMP были использованы для параллельной реализации алгоритмов?**

В ходе работы использовались следующие директивы OpenMP:

`#pragma omp parallel for` - для распараллеливания выполнения циклов;

`#pragma omp parallel` - для создания параллельной области;

`#pragma omp critical` - для защиты общих данных при объединении результатов;

`reduction` - для корректного объединения частичных результатов, вычисленных в разных потоках.

**4. Какие преимущества и недостатки параллельной реализации алгоритмов сортировки на CPU?**

Преимущества:
-	ускорение выполнения алгоритмов на многоядерных процессорах;
-	более эффективное использование вычислительных ресурсов CPU;
-	уменьшение времени выполнения для больших массивов данных.

Недостатки:
-	накладные расходы на создание и синхронизацию потоков;
-	сложность реализации и отладки параллельного кода;
-	не все алгоритмы сортировки хорошо поддаются параллелизации.

**5. Как можно измерить производительность программы в C++?**

Производительность программы можно измерить с помощью библиотеки `<chrono>`.
Для этого фиксируется время начала выполнения участка кода и время его завершения, после чего вычисляется разница между ними. Полученное время позволяет оценить скорость работы алгоритма и сравнить последовательные и параллельные реализации.

**6. Как изменяется производительность сортировок при увеличении числа потоков?**

При увеличении числа потоков производительность улучшается за счёт параллельного выполнения операций. Однако рост ускорения ограничен:
-	количеством доступных ядер процессора;
-	накладными расходами на синхронизацию потоков;
-	особенностями самого алгоритма.
После определённого числа потоков дальнейшее увеличение может не давать заметного прироста производительности.

**7. В каких ситуациях параллельная сортировка может быть менее эффективной, чем последовательная?**

Параллельная сортировка может работать медленнее последовательной в следующих случаях:
-	при небольшом размере массива;
-	при высокой стоимости синхронизации между потоками;
-	при использовании алгоритмов с сильной зависимостью между шагами (например, сортировка вставкой);
-	при недостаточном количестве вычислительных ядер.

В таких ситуациях накладные расходы на параллелизм превышают получаемое ускорение.
